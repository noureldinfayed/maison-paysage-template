Maison Paysage

Premium Landscaping Next.js Template — Documentation

Next.js 15+ TypeScript Tailwind CSS Three.js GSAP Framer Motion Resend

1. Introduction

Maison Paysage is a cinematic, high-end landscaping and outdoor design template built for agencies, landscape architects, and premium service providers.

What's included:

  • Full Next.js 15+ source code (TypeScript)
  • Three.js animated depth background
  • GSAP scroll animations throughout
  • Framer Motion micro-interactions
  • Contact form with Resend email integration
  • Complete documentation (this file)
  • GPT Image 2 generated demo images (commercially licensed, free to use and modify)

2. Requirements

  • Node.js 18+ (LTS recommended)
  • npm or yarn package manager
  • A Vercel account (free tier works) for deployment

3. Installation

1
Extract the ZIP file to your preferred location.
2
Open a terminal in the project folder.
3
Install dependencies:
npm install
4
Copy the environment file:
cp .env.example .env.local
5
Fill in your environment variables in .env.local (see section 4.7 for Resend setup).
6
Start the development server:
npm run dev
7
Open http://localhost:3000 in your browser.

4. Customization

4.1 Changing Colors

All colors are controlled via CSS custom properties in app/globals.css:

:root {
  --color-primary: #d5b46b;    /* Warm gold — buttons, accents */
  --color-secondary: #ffffff;  /* White — text on dark */
  --color-accent: #d5b46b;     /* Accent — selection, scrollbar */
  --color-background: #000000; /* Page background */
  --color-text: #ffffff;       /* Default text color */
}

Change --color-primary from the warm gold (#d5b46b) to any hex value to transform the entire color scheme. The gold accent color is also referenced directly as #d5b46b in some component styles — search and replace if you want a different accent.

4.2 Changing the Logo

The logo placeholder is located in app/page.tsx inside the SiteHeader component.

Replace the SVG placeholder with your own logo:

<img src="/images/logo.png" alt="Your Company" height="32" />

Or paste your own SVG code directly in place of the placeholder SVG.

4.3 Changing Images

All images are in the /public/images/ folder. Replace any image file with your own — keep the same filename to avoid broken references.

Image files used:

  • service-01.jpg, service-02.jpg, service-03.jpg — Service cards
  • project-01.jpg through project-05.jpg — Project cards
  • process-01.jpg through process-04.jpg — Process step backgrounds
  • terrace/main.jpg — Hero background/preload image

All demo images were generated with GPT Image 2 and are commercially licensed — free to use and modify.

4.4 Changing Text Content

All text content lives in app/page.tsx. The page is organized into clearly labeled data arrays at the top of the file:

  • STATS — Hero statistics
  • SERVICE_CARDS — Services section
  • PROJECT_CARDS — Projects section
  • PROCESS_STEPS — Approach/process section

Find the text string you want to change and edit it directly. No CMS required.

4.5 Three.js Background Scene

The animated parallax depth background is in components/DepthBackground.tsx. It creates a subtle mouse-tracking and scroll-reactive depth effect using CSS transforms driven by vanilla JS (no heavy Three.js scene graph).

To adjust the parallax intensity: find the multiplier values (e.g., 0.32) and increase/decrease them.

To disable the background: remove the <DepthBackground /> component from app/page.tsx and optionally replace with a static gradient or image.

4.6 GSAP Scroll Animations

Scroll animations are handled by GSAP ScrollTrigger in the useEffect block of app/page.tsx.

Key configuration points:

  • Animation speed: Change the duration property (in seconds). Default is 0.8 for sections, 0.55 for cards.
  • When animation triggers: Change the start property in ScrollTrigger config. Example: start: "top 82%" means the animation triggers when the element's top reaches 82% of the viewport height.
  • Stagger delay: The stagger: 0.1 value controls the delay between each card's entrance animation.

4.7 Contact Form Setup

The contact form uses Resend to send emails.

1
Create a free account at resend.com
2
Get your API key from the Resend dashboard
3
Add to .env.local:
RESEND_API_KEY=re_your_api_key_here
CONTACT_EMAIL=your@email.com
4
Restart the dev server: npm run dev

The form will now send submissions to your email address. Rate limiting (3 requests per 10 minutes per IP) is built in.

5. Deployment on Vercel (Recommended)

1
Push your project to a GitHub repository.
2
Go to vercel.com and import your repository.
3
In Project Settings → Environment Variables, add your .env.local values.
4
Click Deploy.
5
Your site will be live in ~60 seconds.

6. Changing the Language

The template ships in English. To add French (or any language):

  1. Install next-intl: npm install next-intl (already included)
  2. Create translation files in /messages/en.json and /messages/fr.json
  3. See next-intl documentation: next-intl-docs.vercel.app

Or simply edit the text strings directly in app/page.tsx — all content is in one file for easy editing.
